Documentation pages are plain MDX files. This page is
`content/docs/getting-started.mdx`.

## Adding a page

Create a new `.mdx` file under `content/docs/`. The frontmatter drives the
title and description:

```mdx
---
title: My New Page
description: A short summary shown in search and metadata.
---

Your content here.
```

The file's path becomes its URL — `content/docs/guides/setup.mdx` is served at
`/docs/guides/setup`.

## Ordering & folders

Fumadocs derives the sidebar from the folder structure. To control ordering or
add section labels, drop a `meta.json` alongside your pages:

```json
{
  "title": "Guides",
  "pages": ["setup", "advanced"]
}
```

## Markdown features

Everything you'd expect works out of the box — headings, tables, code blocks
with syntax highlighting, and fumadocs components like callouts:

<Callout type="info">
  This is an info callout rendered by fumadocs.
</Callout>
